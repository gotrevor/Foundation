## 🧠 Context to carry forward
- **The single most useful fact:** v4.31's dominant break is **pattern F** — `simpa`'s final
  acceptance check became *syntactic* instead of up-to-defeq. So hundreds of `simpa using h` where `h`'s
  type is defeq-but-not-syntactic to the goal now error "Type mismatch: After simplification". Fix
  ladder (try in order): **`exact h`** (pure defeq) → **`have h := term; simp [opts] at h; exact h`**
  (the *term* needs simp normalization, e.g. unpacking ∃ / unfolding a def / Finset reindex) →
  **`simp only [opts]; exact h`** (the *goal* needs simp). This one substitution cleared ~40 of the ~60
  files. The cookbook `~/src/mathlib-bump-cookbook/v4.29.1-to-v4.31.0.md` + my staged
  `COOKBOOK-ADDITIONS-v4.31.md` (patterns P–U) document every variant — READ them at lap start.
- **Second pattern:** `grind` no longer whnf-reduces a `match` on concrete `Sum`/`Formula`
  constructors. Symptom: a relation defined by `match … with | .inl,.inr => …` stays opaque in grind's
  e-graph, or grind can't even prove `Sum.inl 0 = Sum.inl 0`. Fix: `simp only [Frame.Rel', Frame.root,
  default]` to force iota, then grind/trivial/rfl — or an explicit witness proof. (Hit in ExtendRoot,
  Rank, KHen, MDP, Unnecessitation, VF, NB/WF, S4Point2.) Beware: `🡒`/`⋏` are typeclass projections
  (`Arrow.arrow`/`Wedge.wedge`), NOT the raw `Formula.imp`/`.and` of match patterns — unfold those
  (+`Formula.top_def` for `⊤`) before iota fires.
- **Third pattern:** v4.31 rejects an `instance` whose return type isn't a class (`Set.Nonempty`,
  `Set.Countable`) or that has a non-inferable explicit arg → make it `lemma`/`def`. The conditional
  `buildAxioms.instHasJ*` ones must be `def` (a macro consumes them as terms). (Hull, LoewenheimSkolem,
  Minimal, Veltman, SubLanguage.)
- **Abandoned/parked, don't redo:** ClassicalEmbedding (l.129/451) — I tried `⟨…⟩`/`exact`/double-wrap;
  the real blocker is the **`⊢ᵀ [emb ψ]` ↔ `𝐋𝐊 ⊢! ψ` definitional bridge** in FirstOrder.Proof, which
  I didn't trace. Universe.lean l.139 — burned time on the `QPF.Fix UniverseFunctor` def-wrapper +
  `Small ↑↑s` instance-synth tangle; `liftp_iff` (l.63) is the right lemma but the coe/instance handling
  needs care (l.132 rec_mk IS fixed). These two are genuinely fiddly — fresh eyes + tracing the
  definitions, not more `simpa` guessing.
